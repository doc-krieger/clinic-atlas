## Interaction Contracts

### PDF Upload (SRCI-01, SRCI-03)

**Drop zone:**
- Default: Dashed border (`border-dashed border-border`), 160px min-height, centered icon + text
- Drag over: Border changes to accent (`border-primary`), background tints (`bg-primary/5`)
- Accepts: `.pdf` only, max 50 MB (D-03)
- Click anywhere in drop zone opens native file picker

**Upload flow:**
1. File selected -- validate size client-side. If > 50 MB, show inline error below drop zone (no modal)
2. Upload starts -- drop zone collapses, `IngestionProgress` appears with SSE-driven progress bar
3. Progress events: "Uploading...", "Parsing page N/M", "Indexing...", "Complete"
4. On success: progress bar fills to 100%, success icon + "Source indexed" text, source appears in list below
5. On scanned PDF detection (D-01): yellow warning Alert appears inline: "This PDF appears to contain scanned images. Text extraction may be incomplete." Source is still stored but marked with warning badge.
6. On duplicate (D-02): inline error: "This document has already been ingested." with link text to existing source.

**Title editing:**
- After successful ingestion, source title appears as editable inline text (click to edit, Enter to save, Escape to cancel)
- Pre-filled from PDF metadata or filename (D-04)

### URL Fetch (SRCI-02)

**Input form:**
- Single text input with placeholder "https://cps.ca/..." and "Fetch" button (accent, primary variant)
- Input validates URL format on blur. Invalid URL: red border + "Enter a valid URL" below input
- Button disabled until valid URL entered

**Fetch flow:**
1. Submit -- button shows loading spinner, input disabled
2. SSE progress: "Fetching...", "Extracting content...", "Indexing..."
3. If thin content detected (D-07): yellow warning Alert: "Content appears limited -- the page may require authentication."
4. If JS fallback triggered (D-08): progress text updates to "Retrying with browser rendering..."
5. On success: same as PDF success state
6. On failure (network error, 4xx/5xx): red Alert: "Could not fetch this URL. Check the address and try again."

**Title editing:** Same inline edit pattern as PDF upload (D-09).

### SearXNG Search (SRCI-05, D-10, D-11)

**Search form:**
- Text input with placeholder "Search trusted sources..." and "Search" button
- Below input: small muted text: "Results scoped to trusted source domains"

**Results display:**
- Each result: checkbox + title (link-styled but not clickable-to-navigate) + domain badge + snippet (2 lines max, muted text)
- "Ingest selected" button at bottom of results list. Disabled when no checkboxes checked. Shows count: "Ingest 3 selected"
- Empty results: "No results found. Try different search terms."

**Ingestion of selected results:**
- Clicking "Ingest selected" processes each URL sequentially
- Each result item shows its own progress inline (spinner -> checkmark / warning)
- Overall progress text: "Ingesting 2 of 3..."

### Source List

**Display:**
- Flat list below the active tab content, separated by a horizontal rule
- Each row: source title (body text), type badge ("PDF" / "URL" / "Search"), status badge, relative timestamp
- Status badges: "Indexed" (default/muted), "Warning" (yellow), "Failed" (destructive)
- Empty state: "No sources ingested yet. Upload a PDF, fetch a URL, or search trusted sources to get started."

**Sorting:** Most recent first (by ingestion date).

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Drop zone | `role="button"` with `aria-label="Upload PDF file"`. Keyboard: Enter/Space triggers file picker. |
| File input | Hidden native `<input type="file">` triggered by drop zone interaction. `accept=".pdf"` |
| Tab navigation | shadcn Tabs component handles `role="tablist"`, `role="tab"`, `role="tabpanel"`, arrow key navigation |
| Progress bar | `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `aria-label="Ingestion progress"` |
| Checkboxes | Standard shadcn Checkbox with label association. "Ingest selected" button includes count for screen readers |
| Status badges | `aria-label` on each badge (e.g., "Status: warning - scanned PDF detected") |
| Focus order | Tab bar -> active tab content (input/drop zone) -> action button -> source list |
