# Phase 1 — UI Design Contract

> Visual and interaction contract for the foundation phase frontend skeleton. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Phase 1 delivers a chat layout skeleton with placeholder components — no real interactivity. This contract establishes the visual foundation that all subsequent phases build on.

| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, icon-to-label gap |
| md | 16px | Default element spacing, component internal padding |
| lg | 24px | Section padding, sidebar item height padding |
| xl | 32px | Layout gaps between major regions |
| 2xl | 48px | Sidebar width gutters |
| 3xl | 64px | Page-level vertical spacing |

Exceptions: Sidebar width is 256px (16 * 16). Message input area minimum height is 52px (not a multiple of 4, but standard touch target).

**Source:** Claude's discretion per CONTEXT.md. Standard 8-point scale.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading | 18px | 600 (semibold) | 1.2 |
| Display | 24px | 600 (semibold) | 1.2 |

**Rationale:** 14px body is standard for dense information-display tools (Linear, Raycast, VS Code). 12px labels for sidebar items and metadata. 18px headings for section titles. 24px display for the page/route title only.

**Source:** Claude's discretion per CONTEXT.md. Aligned with Linear/Raycast aesthetic (D-41).

## Color

Dark mode is the default theme (D-41). All hex values below are for dark mode. Light mode uses shadcn's built-in theme inversion via CSS variables and next-themes.

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `hsl(240 10% 4%)` | `--background` | Page background, main content area |
| Secondary (30%) | `hsl(240 6% 10%)` | `--card` / `--sidebar` | Sidebar, cards, message bubbles, input area |
| Border | `hsl(240 4% 16%)` | `--border` | All borders, separators |
| Muted text | `hsl(240 5% 46%)` | `--muted-foreground` | Placeholder text, timestamps, metadata |
| Primary text | `hsl(0 0% 93%)` | `--foreground` | Body text, headings |
| Accent (10%) | `hsl(210 100% 60%)` | `--primary` | Primary CTA buttons, active sidebar item indicator, focus rings |
| Destructive | `hsl(0 72% 51%)` | `--destructive` | Destructive actions only (none in Phase 1) |

**Accent reserved for:** Primary CTA buttons, active/selected sidebar item indicator, focus rings on interactive elements, link text. Never for decorative purposes or large surface areas.

**Rationale:** Near-black background with subtle blue tint matches Linear/Raycast. High contrast text (93% white) on dark surfaces. Blue accent is clinical and professional — avoids the green/teal medical cliche.

**Source:** D-41 (dark mode default, clinical-clean, muted accent, high contrast, Linear/Raycast). Exact values are Claude's discretion per CONTEXT.md.

## Layout Contract

### Chat Skeleton (D-40)

Phase 1 delivers a static layout skeleton. No functional chat — just the visual shell.

```
+------------------+----------------------------------------+
|                  |                                        |
|    Sidebar       |         Main Content Area              |
|    (256px)       |                                        |
|                  |   +----------------------------+       |
|  App title       |   |                            |       |
|  Nav placeholder |   |    Empty state message      |       |
|                  |   |                            |       |
|                  |   +----------------------------+       |
|                  |                                        |
|                  |   +----------------------------+       |
|                  |   | Message input (disabled)    |       |
|                  |   +----------------------------+       |
+------------------+----------------------------------------+
```

| Region | Width | Behavior |
|--------|-------|----------|
| Sidebar | 256px fixed | Dark secondary surface. Contains app title and future nav items. |
| Main content | Remaining width, max-width 768px centered | Houses message list area and input. |
| Message input | Full width of main content, 52px min-height | Disabled placeholder in Phase 1. |

### Route Structure

| Route | Component | Phase 1 State |
|-------|-----------|---------------|
| `/` | `page.tsx` | Redirects to `/chat` |
| `/chat` | `chat/page.tsx` | Renders sidebar + empty chat skeleton |

## Interaction States

Phase 1 has minimal interactivity. Document what exists:

| Element | States |
|---------|--------|
| Theme toggle | Dark (default) / Light via next-themes. Toggle in sidebar footer. |
| Message input | Disabled state only. Cursor: not-allowed. Muted background. |
| Sidebar nav items | Static text only. No hover/active states needed yet. |
| Health indicator | Green dot (all services up) / Red dot (any service down) in sidebar footer. Text label: "All systems operational" or "Services unavailable". |

## Loading & Skeleton States

| State | Treatment |
|-------|-----------|
| Initial page load | Full-page background color render immediately (no white flash). next-themes `attribute="class"` with `defaultTheme="dark"` prevents FOUC. |
| Health check loading | Muted pulsing dot (neutral color) in sidebar footer while health endpoint responds. |

## Accessibility Baseline

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | Primary text on dominant background: minimum 7:1 ratio (WCAG AAA). Muted text: minimum 4.5:1 ratio (WCAG AA). |
| Focus indicators | 2px solid accent color focus ring on all interactive elements. Visible in both dark and light modes. |
| Keyboard navigation | Tab order: sidebar items, then main content, then message input. |
| Reduced motion | Respect `prefers-reduced-motion` — disable any transitions/animations. |
| Semantic HTML | `<nav>` for sidebar, `<main>` for content area, `<footer>` for input area. |
